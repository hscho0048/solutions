# 스터디 운영 규칙

## 1) 공통 규칙
- 주당 5문제 풀이 제출
- 제출은 Fork → PR 방식만 허용
- main 브랜치 직접 push 금지 (운영자 포함)
- PR은 승인 3명 완료 후 merge

---

## 2) 제출 마감 권장
- 매주 **일요일 23:59(KST)**까지 PR 생성 권장
- 늦어도 **월요일 12:00(KST)** 전까지 리뷰 가능 상태로 정리

> 마감이 느슨하면 리뷰가 몰리고, 결국 “리뷰 못하는 사람”이 생깁니다.  
> 마감은 빡세게 잡는 편이 리뷰 품질/완주율이 올라갑니다.

---

## 3) PR 규칙
### PR 제목 권장 포맷
- `[Wxx][GroupA] {GitHubID} - 5 problems`
- `[Wxx][GroupB] {GitHubID} - 5 problems`

### 커밋 규칙(권장)
- 너무 세분화하지 말고 문제 단위로 커밋
- 예시
  - `solve: W01 boj_1234`
  - `solve: W01 pgs_150368`

---

## 4) 리뷰 규칙
- 승인 3명 달성 전에는 merge 금지
- 리뷰는 “정답 여부”뿐 아니라 아래를 봅니다.
  - 시간복잡도/공간복잡도 타당성
  - 예외 처리
  - 변수명/가독성
  - 풀이 접근이 더 단순해질 여지

---

## 5) 머지 규칙
- squash merge 권장
- merge 후 브랜치 삭제 권장
